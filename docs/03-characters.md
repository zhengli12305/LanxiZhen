# 角色设定：溪桥镇六位居民

场景设定是一个只有几十户人的小村庄，公共场所只有五六个（木匠铺、面包店、酒馆、集市广场、水井、镇长宅邸），六户人家的家分散在外围。地点越少，NPC日常路径越容易自然交叉，越容易观察到"偶遇"这种有意思的行为。

## 角色列表

| 角色 | 身份 | 性格 | 隐藏的心事/目标 |
|---|---|---|---|
| 陈伯 | 老木匠 | 沉默寡言、固执、手艺精湛 | 视力退化，怕露馅丢了饭碗，一直瞒着所有人硬撑 |
| 阿亮 | 木匠学徒 | 活泼、爱幻想 | 不想学木工，偷偷画画，梦想去镇外当画师 |
| 温姐 | 面包店老板娘 | 热心、爱八卦 | 丈夫去世后独自撑店，其实很孤单 |
| 小满 | 镇长的女儿 | 叛逆、直率 | 暗恋阿亮，但知道父亲不会同意她跟一个学徒 |
| 老周 | 酒馆老板 | 圆滑世故、消息灵通 | 隐约知道陈伯的秘密，但从不说破 |
| 石头 | 守夜猎人 | 孤僻、寡言、观察力强 | 唯一撞见过陈伯视力问题的人 |

完整的静态数据结构见 `data/npcProfiles.ts`，包括每个角色的 `personality`（喂给AI的性格描述）、`secretGoal`（隐藏动机）、`homeLocationId`、`workLocationId`。

## 关系网络

- **师徒线**：陈伯 → 阿亮，日常互动是教学场景，但阿亮心不在焉
- **暗恋线**：小满 → 阿亮（单向，阿亮不知情）
- **秘密线**：石头 知道陈伯的秘密，两人互动带着心照不宣的微妙感；老周也隐约察觉但更谨慎
- **社交枢纽**：温姐 和 老周 是镇上消息流通的两个节点，几乎每个角色的日程都会路过面包店或酒馆

石头的 `homeLocationId` 和 `workLocationId` 是同一个地点（他自己的小屋），这是故意的——因为他是夜间活动的角色，这个设定本身就会让AI生成日程时自然导向"白天很少出门，傍晚才活跃"的行为模式，不需要额外写规则约束。

## 给AI生成Prompt时的使用方式

生成当日计划时，每个NPC的prompt里应该包含：`personality` + `secretGoal` + `relationships`（尤其是 `note` 字段里那句话），这样生成出来的行为会自然带上人物弧光的痕迹，而不是六条互不相干的平行时间线。**不要**只把`role`喂给AI就完事，那样生成的日程会很扁平，缺少这个项目最有价值的"关系驱动行为"的效果。
